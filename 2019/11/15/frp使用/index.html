    内网穿透 frp

    frp 是一个可用于内网穿透的高性能的反向代理应用，支持 tcp, udp, http, https 协议。 官方中文文档在 这里，项目主页在 这里。

官方文档写得非常详细，建议移步阅读。其区分 服务端 与 客户端 ，以下是简化的部署步骤。

    服务端

    到 `版本页` 进行下载，此处拿最新的。服务端需要搭建在一台具有 固定公网IP 的服务器。

    自行选择相应的平台

    cd /data/ && wget https://github.com/fatedier/frp/releases/download/v0.20.0/frp_0.20.0_linux_amd64.tar.gz

#  解压出来
tar -xf frp_0.20.0_linux_amd64.tar.gz
cd frp_0.20.0_linux_amd64

    修改 frps.ini 服务端设置

    cat > frps.ini <<EOF
[common]
bind_port = 7000
privilege_token = 112233445566
privilege_mode = true
EOF

# 编辑启动服务文件：
cat >/etc/systemd/system/frps.service <<EOF
[Unit]
Description=frps daemon

[Service]
Type=simple
ExecStart=/data/frp_0.20.0_linux_amd64/frps -c /data/frp_0.20.0_linux_amd64/frps.ini

[Install]
WantedBy=multi-user.target
EOF

# 启动服务即可
 sudo systemctl start frps.service
 sudo systemctl enable frps.service

    客户端

    客户端即是公司内部的服务器，同样选择在一台 ubuntu 系统来部署。

    # 客户端配置文件
cat > /data/frp_0.20.0_linux_amd64/frpc.ini <<EOF
[common]
server_addr = x.x.x.x # 服务端的公网IP
server_port = 7000
privilege_token = 112233445566

# 以下是新添加的
[ssh]
type = tcp
local_ip = 127.0.0.1
local_port = 22
remote_port = 9167

EOF

    测试

    到此，服务端 与 客户端的配置已经完成。
    注意，服务端的防火墙要允许7000端口访问，如果有额外开其他端口，也要一并放行。

    假设从外网连接 公司内网 客户端的 ubuntu 服务器，则可以添加以下设置。 

    # 重启客户端服务

 sudo systemctl restart frpc.service
# 接着从外网访问即可。

 ssh -p 9167 user@x.x.x.x

    如无错误，已经可以访问内部服务器
