    kylin系统下架设iscsi服务与使用

    iscsi是一种通过ip网络使用的块存储，通过服务端直接给其他客户端挂载使用，如同本机磁盘。

    系统基础环境

    1. 服务端

      • OS：Kylin 4.0.2
      • 硬盘：
        • sda 20 G
        • sdb 50 G （系统根目录）
      • 网络IP： 192.168.220.137

    2. 客户端

      • OS：Kylin 4.0.2
      • 硬盘：
      • sda 50 G （系统根目录）
      • 网络IP： 192.168.220.138

    服务端架设

    安装服务程序

    需要在iSCSI目标服务器上安装Target Framework（TGT）软件包

    apt-get install tgt -y
# 检查tgt状态
systemctl status tgt

    配置iSCSI target

    在iSCSI服务器上创建LUN（逻辑单元号）设备。LUN是启动器将连接到并在以后使用的后端存储设备。

    /etc/tgt/conf.d/iscsi.conf
    <target iqn.2019-11.example.com:lun1>
    backing-store /dev/sdb
    initiator-address 192.168.220.138
    incominguser iscsi-user password
    outgoinguser iscsi-target secretpass
</target>

    backing-store: 本机存储设备作为后端iSCSI target
    initiator-address: 指定客户端ip地址可以访问
    incominguser: 用于保护LUN的传入用户名/密码 （可不加）
    outgoinguser: 用于相互CHAP身份验证的传出用户名/密码。（可不加）

    完成后保存并关闭文件。然后，重新启动TGT服务以应用配置更改：

    systemctl restart tgt

    检查配置是否生效以及符合预期

    tgtadm --mode target --op show

    iscsi客户端

    安装iSCSI

    在iSCSI使用挂载服务器上安装iSCSI启动器软件包。

    apt-get install open-iscsi -y

    • 可以通过修改文件，更改本机客户端的iqn

      vim /etc/iscsi/initiatorname.iscsi
## 修改 InitiatorName=后的值即可
InitiatorName=iqn.1993-08.org.debian:01:1c5a149ad91

    对iSCSI target服务器运行目标发现以找出共享target。

    iscsiadm -m discovery -t st -p 192.168.220.137

    发现target后，会在/etc/iscsi/nodes/下生成对应target的iqn主机信息，同时/etc/iscsi/send_targets下也会生成指向该目录文件的软链接。

    root@zqingfa-kylin:~# tree /etc/iscsi/nodes/

/etc/iscsi/nodes/
└── iqn.2019-11.example.com:lun1
    └── 192.168.220.137,3260,1
        └── default


root@zqingfa-kylin:~# tree /etc/iscsi/send_targets
/etc/iscsi/send_targets
└── 192.168.220.137,3260
    ├── iqn.2019-11.example.com:lun1,192.168.220.137,3260,1,default -> /etc/iscsi/nodes/iqn.2019-11.example.com:lun1/192.168.220.137,3260,1
    └── st_config


vim /etc/iscsi/nodes/iqn.2019-11.example.com\:lun1/192.168.220.137\,3260\,1/default

# 添加修改文件中如下对应的用户名与密码，如果有的话

node.session.auth.authmethod = CHAP  
node.session.auth.username = iscsi-user
node.session.auth.password = password
node.session.auth.username_in = iscsi-target
node.session.auth.password_in = secretpass
node.startup = automatic

    重新启动iSCSI启动器服务以应用配置更改：

    systemctl restart open-iscsi

    • 使用lsblk命令即可查看当前的磁盘状态

    • 在target服务器使用如下命令查看iscsi链接状态

      root@zqingfa-kylin:/home/zqingfa/桌面# tgtadm --mode conn --op show --tid 1
Session: 2
    Connection: 0
        Initiator: iqn.1993-08.org.debian:01:1c5a149ad91
        IP Address: 192.168.220.138

    至此已经完成了iSCSI的部署与挂载使用。
